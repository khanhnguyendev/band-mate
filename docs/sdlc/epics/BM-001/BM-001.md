# BM-001: Auth and Accounts

## Overview
Covers user registration, login, and session management using Supabase Auth with JWT tokens. Implements email and password authentication with mandatory email verification before first login. Provides a global NestJS guard strategy with an opt-out decorator for public endpoints.

## Affected Modules
auth, users, wallet

## Key Decisions
- Supabase Auth used for identity management; JWTs issued and verified by Supabase
- Email + password is the only supported authentication method at launch
- Email verification is required before a user can log in for the first time
- A global NestJS `APP_GUARD` enforces authentication on all routes by default
- `@Public()` decorator opts individual routes out of the global auth guard
